2. Пусть рыцарь получит возможность взять в руки меч. У него же есть возможность владеть мечом – у него есть поле. Теперь нужно добавить действие: взять меч! Сделаем рыцарю метод public void setSword(Sword sword). Он получает какой-то меч в качестве параметра. Пусть он этот полученный в параметре меч сохранит в поле меча в классе Knight. Тогда получится, что мы в setSword будем в круглых скобках передавать объект меча, и этот объект меча будет сохраняться в рыцаря. Т.е. как будто он берет его в руку. Параметры из круглых скобок мы уже сохраняли в поля класса, когда работали с конструкторами: здесь тот же принцип.

   Должно получиться:

    ```java
    public void setSword(Sword sword) {
    	this.sword = sword;
    }
    ```

   Обратите внимание на слово this. Оно показывает, что значение через точку от него является полем класса, а если this не написано, как после знака равно, то это параметр метода. В данном случае this применяется, чтобы разрешить конфликт имен: наш параметр в круглых скобках называется точно так же, как и поле класса - sword. Один из полезных вариантов использования this. В данном случае писать this необходимо, потому что java не в состоянии понять, что именно мы имеем в виду: параметр или поле, и эту ситуацию нам нужно разрешить явно.

5. Ранее в методе main в классе App мы уже создали рыцаря, мага и лучника. Также мы создали объекты меча, посоха и лука. Теперь давайте сделаем, чтобы 4 персонажа (2 рыцаря, лучник, маг) взяли в руки соответствующее оружие, которое мы создали для них. Для этой цели мы как раз и создали методы set… Вызываются эти методы так же, как и hit: пишем имя объекта, у которого хотим вызвать метод (кого хотим попросить взять в руки оружие), точка, название метода (помните, что у каждого класса методы называются по-разному) и круглые скобки, а в круглых скобках мы как раз пишем имя объекта оружия, который мы хотим дать персонажу. Обобщенная запись:

    ```java
    character.setWeapon(weapon);
    ```

   Обратите внимание, что вызывать метод объекта, и использовать объект в качестве параметра можно лишь тогда, когда этот объект уже создан. Т.е. писать эту строку можно только ниже создания объектов персонажа и его оружия.

   Кроме того, для каждого персонажа нужно создать отдельный объект оружия. Например, два рыцаря не могут владеть одним мечом одновременно – это нелогично. Т.е. нам понадобится два меча, один посох и один лук.

7. Поработаем с навыками. Перейдем в класс Heal. Это класс навыка лечения. Им может владеть только маг (полю characterClass выставим значение по умолчанию «Wizard»). Давайте опишем в этом классе, как действует лечение. Для этого создадим в нем метод: `public void apply(Knight knight)`

   Т.е. наш метод называется apply (применить), а в качестве параметра он принимает объект того, кого нужно вылечить. В данном случае этот навык пока лечит только рыцарей. Теперь как он должен работать, т.е. что должно происходить внутри фигурных скобок этого метода: пусть при вызове метода тому, кого нужно вылечить, прибавляется 10 хп.

   Тот, кого нужно лечить, – параметр в круглых скобках метода. Прибавляем ему хп так же, как и отнимали до этого в методе hit. Только минус нужно заменить на плюс.

   Кроме того, пусть на экран выводится сообщение о том, что такой-то рыцарь был вылечен, и теперь у него столько-то хп.

11. Пока мы делаем 2д игру, поэтому чем описывается точка в двумерном пространстве? Конечно, координатами. Сделайте в классе Point поля для координат
    - тип: целое число,
    - названия: x, y

    конструктор, который принимает значения координат в качестве параметров и сохраняет их в поля, и три метода public void setX, public void setY, public void setXY, каждый из которых принимает соответствующее своему названию количество параметров и сохраняет эти значения в поля. Эти методы нужны для того, чтобы мы могли создать точку с помощью конструктора с какими-то координатами, а потом, если нам нужно изменить координаты, мы могли просто вызвать метод объекта, а не создавать новый объект.

    В конструкторе также можно использовать слово this для указания полей. Тогда параметры конструктора могут иметь такие же имена, что и поля класса. Эта запись является предпочтительной, т.к. названия в этом случае являются намного более говорящими о своем смысле. Т.е. Point(int x, int y), а в внутри фигурных скобок к полям обращаемся через this, а к параметрам конструктора (в круглых скобках) просто как есть.
